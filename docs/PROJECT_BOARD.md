# Project Board Workflow & Roadmap

## Workflow Explanation

The project board follows a linear progression to ensure transparency and quality control throughout the development lifecycle.

- **Backlog** (Not yet started): Ideas, requested features, and reported bugs that have been triaged but are not yet scheduled for immediate development.
- **Building** (Actively being implemented): Tasks currently being worked on by a developer.
- **Review** (PR open, under review): Implementation is complete, and a Pull Request has been opened. The task is awaiting peer review and approval.
- **Shipped** (Merged to dev or prod): The code has been merged into the target branch and is considered complete.

## Roadmap Suggestion

### Phase 1: Core Sync & Auth
- Implementation of robust authentication systems.
- Establishing core data synchronization patterns.
- Initial project structure and infrastructure setup.

### Phase 2: Video/Audio via Daily.co
- Integration of Daily.co for real-time communication.
- Implementation of video and audio calling features.
- UI/UX for communication overlays and controls.

### Phase 3: Beta Launch & Polish
- Final bug scrubbing and performance optimization.
- Beta testing with a limited user group.
- Final polish and official launch.
